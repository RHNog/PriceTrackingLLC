# Whitepaper Template

Use this template for deeper strategic, technical, or market-facing papers.

## Structure

1. Title
2. Executive Abstract
3. Problem Context
4. Project Phronesis Position
5. Current Platform
6. Future Direction
7. Architecture Or Operating Model
8. Evidence And Constraints
9. Risks And Limitations
10. Conclusion
11. Appendix

## Rules

- Explain why each decision matters.
- Separate evidence from interpretation.
- Separate current capability from future vision.
- Avoid proprietary internals unless the whitepaper is internal-only.
- Use diagrams for relationships and process.
- Use tables only when comparison improves comprehension.
