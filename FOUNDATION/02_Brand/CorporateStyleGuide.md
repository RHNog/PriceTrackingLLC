# Project Phronesis Corporate Style Guide

This guide defines the corporate document system for Project Phronesis proposals, summaries, partner materials, and investor-facing documents.

Project Phronesis is the internal engineering initiative responsible for developing an evidence-driven decision intelligence platform for collectible markets. The commercial brand remains intentionally undecided.

## Purpose

The corporate document system should make Project Phronesis materials feel:

- Minimal.
- Elegant.
- Premium.
- Professional.
- Executive.
- Honest.
- Evidence-based.
- Forward-looking.
- Calm.
- Premium without ornament.

The style should support serious partnership conversations. It should not feel like marketing collateral, a product landing page, or a speculative pitch deck.

Avoid:

- Corporate clipart.
- Decorative illustrations.
- Heavy borders.
- Dense ornamental graphics.
- Visual clutter.

## Core Principle

Prefer clarity over decoration.

Prefer whitespace over visual noise.

Prefer defensible statements over persuasive exaggeration.

Every page should communicate one primary idea.

## Logo Usage

Use the approved Project Phronesis raster logo when available.

The template reserves a fixed logo area so the current raster logo can later be replaced by a vector version without changing the document layout.

Logo placement:

- Cover page: centered above `PROJECT PHRONESIS`.
- Header: left aligned.
- Minimum clear space: at least the height of the logo mark on all sides.
- Do not stretch, recolor, distort, crop, rotate, or add effects.

If the approved raster logo is unavailable in the repository, use the reserved logo placeholder and replace it before external distribution.

## Visual Identity

### Cover Page

The cover page must include:

- Centered logo.
- `PROJECT PHRONESIS`.
- `Practical Wisdom. Better Decisions.`
- Document title.
- Prepared For.
- Date.
- Version.

The cover should feel quiet and authoritative. Avoid decorative shapes, background gradients, dense imagery, and oversized claims.

### Header

Every page after the cover should include:

- Project Phronesis logo.
- `Project Phronesis`.
- Document title.
- Thin separator line.

### Footer

Every page after the cover should include:

- `Confidential`.
- `Project Phronesis`.
- `Page X of Y`.

## Typography

### Headings

Use `Cinzel` for headings.

Fallbacks:

- Georgia.
- Times New Roman.
- Serif.

Heading style:

- Title Case.
- Controlled spacing.
- No negative letter spacing.
- Avoid oversized type inside dense sections.

### Body

Use `Inter` for body copy.

Fallbacks:

- Arial.
- Helvetica.
- Sans-serif.

Body style:

- Clear paragraph rhythm.
- Comfortable line height.
- Short sections.
- Bullets only when they improve scanning.

## Color Palette

| Token | Hex | Usage |
| --- | --- | --- |
| Deep Navy | `#0B1F33` | Primary text, headers, footer text |
| Gold | `#C6A15B` | Accent line, restrained emphasis |
| Slate | `#5D6B7A` | Secondary text |
| White | `#FFFFFF` | Page background |
| Off White | `#F7F4EE` | Optional cover background or subtle section field |

Use minimal color. Gold should be an accent, not a dominant field.

## Layout System

### One Idea Per Page

Each page should have one primary communication job. If a page contains two unrelated ideas, split it.

Recommended page types:

- Cover.
- Executive summary.
- Strategic thesis.
- Marketplace value.
- Flywheel diagram.
- Regional vision.
- Workflow map.
- Responsible platform philosophy.
- Alignment model.
- Boundary statement.
- Platform snapshot.
- Call to conversation.

### Whitespace

Use large margins and generous vertical spacing. The document should feel like an executive brief, not a dense memo.

### Callout Panels

Use callout panels for:

- Platform Vision.
- Marketplace Alignment.
- Community Value.
- Responsible API Philosophy.
- Global Vision.
- Key messages.

Callout panels should use off white background, deep navy text, and a restrained gold accent line.

### Diagrams

Use simple vector-style diagrams whenever a diagram communicates faster than prose.

Preferred diagrams:

- Marketplace flywheel.
- Regional ecosystem map.
- Future workflow chain.
- Platform snapshot.

Diagram rules:

- Use simple boxes, arrows, and labels.
- Use Deep Navy outlines and Gold accents.
- Avoid decorative icons and illustrations.
- Keep labels short.
- Explain the diagram with one short caption.

### Tables

Use comparison tables for:

- Current Platform vs Future Vision.
- Marketplace benefits.
- Community benefits.
- What Project Phronesis is not.

Tables should have light rules, generous cell padding, and no heavy borders.

## Proposal Structure

Every corporate proposal should contain:

1. Cover.
2. Executive Summary.
3. Why Project Phronesis Exists.
4. Why This Benefits The Partner.
5. Marketplace Flywheel.
6. Global Ecosystem Vision.
7. Future User Workflows.
8. Responsible Platform Philosophy.
9. Affiliate Or Incentive Alignment, when relevant.
10. What Project Phronesis Is Not.
11. Long-Term Partnership Vision.
12. Call To Conversation.
13. Platform Snapshot.
14. Appendix.

## Writing Standards

Write in a professional, executive voice.

Use language that can be comfortably defended in a technical or business conversation.

Every proposal should:

- Clearly separate current capabilities from future vision.
- Explain ecosystem value without promising outcomes.
- Avoid exaggerated technical claims.
- Avoid implying partner approval, access, or endorsement.
- Avoid exposing confidential implementation details.
- Use `Project Phronesis` as the engineering initiative, not the commercial brand.
- Reduce walls of text with callouts, diagrams, tables, and large headings.
- Preserve a calm executive voice.

## Disclosure Rules

Partner-facing corporate documents may discuss:

- Vision.
- Mission.
- Business philosophy.
- Responsible integration.
- Community value.
- Marketplace alignment.
- Future workflow direction.

Partner-facing corporate documents must not disclose:

- Proprietary reasoning methods.
- Internal implementation details.
- Sensitive data assets.
- Evaluation methods.
- Ranking or weighting systems.
- Competitive intellectual property.
- Security-sensitive credentials or operational secrets.

Use this system for future documents addressed to:

- PSA.
- Cardmarket.
- Beckett.
- eBay.
- Investors.
- Marketplace partners.
- Data providers.
- Strategic business partners.

For each audience, preserve the same visual system and disclosure rules. Adapt only the value narrative, partner-specific ecosystem alignment, and requested next conversation.

## Executive Review Standard

Before external distribution, review the finished document as if you were:

- A Fortune 500 executive.
- A technology investor.
- A marketplace VP.
- An engineering director.

Ask:

- Does the document immediately communicate professionalism?
- Does it encourage a meeting?
- Does it reflect the quality of the platform?
- Does it avoid over-claiming?
- Does each page communicate one primary idea?
- Are current capability and future vision clearly separated?

If the answer to any question is no, refine the presentation before sending.
