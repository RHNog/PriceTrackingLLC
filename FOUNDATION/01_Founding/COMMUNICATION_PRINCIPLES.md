# Communication Principles

Project Phronesis communicates with restraint.

The organization should never need hype to sound serious.

## Core Belief

Communication should make decisions easier.

This matters because Project Phronesis serves people making real decisions under uncertainty. If communication exaggerates, decorates, obscures, or over-promises, it weakens the trust the platform is trying to build.

## Communication Principles

### Clarity Over Impressiveness

Do not write or design to impress the reader with complexity.

Every sentence, diagram, callout, and page should remove an obstacle between the reader and the core message.

### Current Capability Before Future Vision

Always separate what exists now from what is planned, possible, or aspirational.

Future vision is valuable, but only when the reader can tell it is future vision.

### Defensible Claims Only

Every technical, business, or partnership claim should be comfortable to defend in a serious conversation.

If a statement over-promises, weakens trust, or implies unsupported maturity, rewrite it.

### Executive Materials Should Respect Time

Partner, investor, and executive materials should be understandable quickly.

Use one primary idea per page. Use diagrams only when they clarify. Use whitespace as a comprehension tool. Avoid walls of text, dense tables, and decorative graphics.

### Technical Materials Should Preserve Detail

Technical documentation should be implementation-grade.

It should explain ownership, boundaries, edge cases, dependencies, failure behavior, and testing expectations well enough that a future engineer or AI agent can continue the work.

### Public Value, Private Machinery

External materials may explain vision, mission, business philosophy, responsible integration, community value, marketplace alignment, and future workflow direction.

They must not expose proprietary reasoning methods, internal implementation details, sensitive data assets, evaluation methods, ranking systems, weighting systems, credentials, or competitive intellectual property.

## Visual Communication

Project Phronesis should look the way it thinks:

- Minimal.
- Calm.
- Premium.
- Evidence-based.
- Restrained.
- Clear.

Visual systems should use Deep Navy, Gold, Slate, White, and Off White with minimal ornament.

Design exists to clarify. It should not become performance.

## Communication Standard

Before publishing or sending, ask:

- What is the reader supposed to understand?
- What can be removed?
- What is current capability?
- What is future vision?
- What might be mistaken as a promise?
- What should remain internal?
- Does the design make the message easier to understand?
