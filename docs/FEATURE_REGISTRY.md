## Registry Rule

Every meaningful future feature, workflow, architecture decision, technical system, or product-facing capability must receive a permanent Feature ID before or during implementation.

Feature IDs must appear in:

- Feature specification.
- Implementation prompt.
- Validation notes.
- Release notes.
- Roadmap or sprint history when relevant.
- Atlas when the feature changes project knowledge.
